The problem

A tower with no radar.

Claude Code is easy to run — and easy to run a lot. The trouble isn't any one agent. It's the loss of the room: no single place that answers who needs you, why something stopped, and where you actually are.

Lost in the tabs

You can't see who needs you

Six terminals, six agents. One has been waiting on an approval you never saw. One failed twenty minutes ago. Two are editing the same file. You are the control tower — but you're flying blind.

Is it me or Anthropic?

An error, and no way to place it

Claude stops mid-turn. Your Wi-Fi? A slow hotel network? Or Anthropic having a moment? You alt-tab to a speed test and guess — and guessing wrong wastes the next ten minutes.

The silent VPN hop

Location drifts, and the turn dies

You pin Claude to a country. Your VPN silently hops. Now a request fails with a confusing Please run /login-looking error instead of simply waiting for you to come back on-country.

Blind to the meter

You don't know your real usage

"How much of my plan is left?" You don't actually know until you hit the wall mid-task. Local token math is a guess; the real number lives behind claude /usage.

What Tower does

Four jobs, one daemon.

One background process holds all the logic and state. Everything you see — in the menu bar or the terminal — is a thin, honest view over it.

Traffic

The agents

Every running Claude agent, live: model, effort, status, and momentum. A ranked needs-you queue — failed > blocked > asking > done — with notifications, and click-to-focus that jumps to the agent's terminal.

Weather

The weather

When Claude errors mid-session, Tower answers the only question that matters — is it my internet, is it just slow, or is it Anthropic? — from passive latency probes, plus an on-demand speed test.

Fence

The fence

Zero-dependency curses. Now with a mouse.

Pure Python stdlib — the same state.json and command files as the app. Deliberately non-reactive so you never fat-finger routing off, but everything's clickable: toggle the guard, pick a country, run a speed test, focus an agent's terminal.

↺
Same brain, other face.Change a setting in one, the other reflects it in about a second.
⌕
Answers on sight.A red header — INTERNET DOWN, WI-FI LOGIN REQUIRED, ANTHROPIC API ISSUE — means stop and look here.
◱
Only repaints on change.~80 ms input, ~60 ms daemon confirms, no idle cost.
The marks

One living glyph per model.

Deliberately monochrome and minimal — still at rest, and while its agent works, each comes alive in its own way. The tier's color appears only in the small label beside it, never on the mark itself.

The philosophy

An honest status layer.

If something looks wrong — off-country, an unstable net, a blocked request — that is the app doing its job and reporting the truth. Not the app being broken. There's nothing to quit; leave it running and it clears itself when your location or connection recovers.

⛔
Fail-closed, never fail-confusing.Claude is allowed only when confirmed in-country and the network has a usable path to Anthropic. Anything uncertain is held.
↩
It always un-routes on shutdown.Quitting removes the proxy from settings.json first, so Claude returns to a direct connection. A crashed UI can't break Claude.
◐
Never through the shell.Routing edits settings.json only — passive, reversible, backed up before the first edit.

Flight rule · a block is pending, not failed

✗
403 would kill the turn. Claude reads it as broken auth → "run /login".
✓
So Tower answers 503 + Retry-After. Held a few seconds, re-checking, so a blip clears with no visible retry.
↻
Claude shows its own spinner. "Retrying · attempt x/y" — the durable pending UX.
▲
It resumes on its own. The moment the guard clears — no lost work, no confusing error.
Inbound

On the radar.

Tower is open source — use it, fork it, port it. Here's what's approaching. One portable Python daemon with thin front-ends is what makes each of these tractable.

Next · planned

Windows tray app

~80% of the code — the daemon — is already portable Python. A C#/.NET NotifyIcon tray plus an ANSI terminal UI, driven by the same JSON. Routing ports for free: Claude Code reads the same settings.json everywhere.

Planned

Linux front-end

The daemon is already stdlib-portable. A tray via AppIndicator and the existing curses dashboard bring the same control tower to a third platform.

Idea

Deeper collisions

Beyond same-repo and same-file: surface the exact lines two agents are about to fight over, and a one-tap "pause the other" so a merge conflict never happens.

Idea

Usage history & forecasts

A longer memory for plan usage — weekly burn-down, a forecast of when you'll hit a limit at your current pace, and a gentle heads-up before you do.

Idea

More fences

Beyond country: time-of-day windows, per-project rules, and allowlists — the same fail-closed proxy, more ways to draw the fence.
